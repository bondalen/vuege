# Дневник разработки Vuege

## [2025-08-13] - Анализ архитектуры vuega и планирование Vuege

### Наблюдения
- Изучен репозиторий vuega: https://github.com/bondalen/vuega.git
- Архитектура vuega: Spring Boot 3.4.4 + Java 24 + MS SQL Server + Vue.js 3.4.18 + Quasar 2.16.0 + Apollo Client
- Используется монолитная архитектура с единым JAR-артефактом
- Frontend собирается через frontend-maven-plugin и копируется в static папку
- GraphQL API с CORS поддержкой
- Реактивный стек: WebFlux + R2DBC

### Решения
- **Java 21** вместо Java 24 (стабильность и совместимость)
- **PostgreSQL 16** вместо MS SQL Server (открытость и ГИС-поддержка)
- **TypeScript** вместо JavaScript (типобезопасность для сложной доменной модели)
- **TanStack Router** вместо Vue Router (полная реактивность)
- **Liquibase** для миграций (версионирование схемы БД)
- **Обновленные версии**: Spring Boot 3.4.5, Vue.js 3.4.21, Quasar 2.16.1

### Проблемы
- Необходимость настройки SSH доступа к приватному репозиторию vuega
- Сложность доменной модели требует строгой типизации
- Исторические данные (4000 лет до н.э.) требуют специальной обработки
- ГИС-функциональность требует PostGIS интеграции

### Технические детали
- PostgreSQL DATE поддерживает диапазон 4713 BC до 5874897 AD
- PostGIS обеспечивает пространственные типы данных
- TanStack Router требует TypeScript
- Liquibase обеспечивает версионирование схемы БД

### Архитектурные решения
- **Реактивный стек**: от БД (R2DBC) до UI (Vue.js + TanStack Router)
- **Единый контейнер**: postgres-java-universal с JAR-артефактом
- **Type-safe API**: GraphQL с TypeScript
- **ГИС-интеграция**: PostGIS + пространственные индексы
- **Исторические данные**: PostgreSQL DATE + валидация

### Следующие шаги
1. Создание полной документации проекта
2. Настройка базовой инфраструктуры
3. Создание доменной модели
