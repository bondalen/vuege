# Основная документация проекта Vuege

Эта папка содержит четыре основных документа проекта Vuege, которые являются ключевыми для понимания и развития проекта.

## Документы

### 📋 [project.md](./project.md)
**Основной документ проекта** - полное описание архитектуры, технологического стека, доменной модели и технических решений проекта Vuege.

**Содержит:**
- Архитектуру проекта
- Технологический стек
- Доменную модель
- API спецификации
- Инфраструктуру

### 📝 [diary.md](./diary.md)
**Дневник разработки** - хронологический журнал всех технических решений, наблюдений и планов в процессе разработки.

**Содержит:**
- Наблюдения и анализ
- Принятые решения
- Технические детали
- Архитектурные решения
- Планы на будущее

### 🔄 [changelog.md](./changelog.md)
**Журнал изменений** - хронологический список всех значительных изменений в проекте с описанием добавленных, измененных и исправленных функций.

**Содержит:**
- Даты изменений
- Описание изменений
- Технические детали
- Версии и релизы

### ❓ [qa.md](./qa.md)
**Вопросы и ответы** - сборник часто задаваемых вопросов и ответов по проекту, архитектурным решениям и техническим аспектам.

**Содержит:**
- FAQ по проекту
- Архитектурные решения
- Технические вопросы
- Проблемы и решения

## Структура документации

```
docs/
├── main/           # Основная документация
│   ├── project.md  # Архитектура и описание проекта
│   ├── diary.md    # Дневник разработки
│   ├── changelog.md # Журнал изменений
│   └── qa.md       # Вопросы и ответы
└── others/         # Вспомогательная документация
    ├── tasktracker.md
    ├── ssh-setup.md
    ├── date-helper.md
    └── ...
```

## Рекомендации по работе

1. **Начинайте с project.md** - для понимания общей архитектуры
2. **Изучайте diary.md** - для понимания процесса принятия решений
3. **Следите за changelog.md** - для отслеживания изменений
4. **Консультируйтесь с qa.md** - для поиска ответов на вопросы

## Обновление документации

При внесении изменений в проект обязательно обновляйте соответствующие документы:

- **Новые функции** → changelog.md
- **Архитектурные изменения** → project.md
- **Технические решения** → diary.md
- **Новые вопросы** → qa.md

## Правило размещения новых документов

**Все новые файлы документов размещаются в папке `docs/others/`**

### Принципы:
- Основные документы (`docs/main/`) остаются неизменными
- Все дополнительные документы добавляются в `docs/others/`
- При добавлении нового документа обновляется README в `docs/others/`
- Ссылки на новые документы добавляются в соответствующие README файлы
